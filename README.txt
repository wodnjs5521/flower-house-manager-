파일 안내

1. index.html
   GitHub Pages에 업로드할 최신 실행 파일입니다.

   React JSX 원본 코드입니다.

   지금까지 단계별로 반영한 기능과 변경 기록입니다.

사용 방법

- 컴퓨터에서 확인: index.html을 더블클릭합니다.
- GitHub Pages 갱신: 저장소의 기존 index.html을 이 파일로 교체합니다.
- 수정 작업을 계속할 때는 ZIP 전체를 보관하고 Codex에 다시 전달하면 됩니다.

주의

- 병해충 예방 날짜와 충태는 정식일, 예상 생육단계, 날짜와 날씨를 이용한 예방용 예측입니다.
- 실제 병해충 발생 확정이나 농약 사용 지시가 아닙니다.
- 농약은 현장 확인 후 PSIS 등록정보와 제품 라벨을 확인해야 합니다.
- 검증된 직접 이미지 주소가 없는 병해충은 참고사진 미등록으로 표시됩니다.

추가된 파일 (2026-06-21 개선)

5. vendor/ 폴더
   React, ReactDOM, Babel을 로컬에 동봉한 폴더입니다. 인터넷 없이도 첫 실행이 됩니다.
   index.html을 올리거나 옮길 때 이 vendor 폴더를 반드시 함께 두세요.

6. firestore.rules
   Firebase 자동 동기화를 쓸 때 권장하는 보안 규칙입니다.
   Firebase 콘솔 → Firestore Database → 규칙에 붙여넣고 게시하세요.

보안·오프라인 안내

- 비밀번호는 더 이상 평문으로 저장하지 않고 해시로 저장합니다. 기존에 쓰던
  비밀번호는 그대로 로그인되며, 로그인할 때 자동으로 해시로 바뀝니다.
- 소스의 Firebase apiKey는 비밀이 아니라 식별자입니다. 실제 보안은 firestore.rules가
  결정하므로, 동기화를 쓰면 위 규칙을 꼭 게시하세요.
- 날씨·꽃 사진·자동 동기화는 인터넷이 필요합니다(없으면 추정 날씨/로컬 저장으로 동작).
